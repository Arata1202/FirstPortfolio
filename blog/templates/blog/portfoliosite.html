開発のきっかけ

2023年の7月にプログラミングの学習を始め、少しずつではありますが知識や技術を習得してきました。

まだまだ実務をこなすほどの実力はありませんが、一つの区切りとして私の努力結果を形にするため、このPortfolioの作成に至りました。

使用言語 / ツール

  • HTML / CSS
  • JavaScript
  • Python / Django
  • SQLite ※MySQLに変更予定
  • AWS / EC2 / Route53
  • Nginx
  • Gunicorn
  • Git / GitHub
  • VSCode

フレームワークはPythonのDjangoを採用しました。

JavaのSpringBootと迷いましたが、現在はPythonの需要が高まってきていることや、将来性や需要の伸びを期待されているということでDjangoを採用しました。

アプリの設計

PWA化

サービスワーカーへ登録し、PWA化しました。ネイティブアプリのようなwebサイトになり、操作性が向上しています。

フッターとスマートフォンのホームバーが干渉しないようなCSSも施しています。

レスポンシブデザイン

PC , スマートフォン , タブレットに対応しています。

ナビゲーションメニュー

ヘッダーを固定しているためタイトルへidを設定してもズレが生じますが、marginとpaddingで微調整し、正確な位置へ遷移できることがポイントです。

ハンバーガーメニュー

スマートフォン、タブレットの場合表示されます。

内部リンクを押した際に、自動でハンバーガーメニューが閉じられるのがポイントです。

スライダー

webサイトをリッチにするため、jQueryでslickのスライダーを採用しました。

内部リンク

内部リンクに気づいてもらえるよう、動作を取り入れて目立つように設計しました。

開発工程

1 , ローカル環境でPortfolioの作成

VSCode , Anaconda Promptを使用してローカル環境で開発しました。


2 , Nginx , GunicornでEC2へデプロイ

初めはDjangoを直接デプロイしましたが、独自ドメインを紐づける事が出来なかったため、こちらの方法を採用しました。


3 , Route 53でドメインを取得し、独自ドメイン化

AWSのRoute53で簡単に紐づけることが出来ました。


4 , SSL化

初めはAWSのACMでSSL化を試みましたが、ロードバランサーの設定が上手くいかず断念しました。

代替案としてNginxでCertbotを使用し、SSL化に成功しました。


5 , 完成

開発期間

約5日かかりました。

上記で述べた工程は成功した事についてしか触れていませんが、失敗もかなり多く、デプロイに丸一日かかってしまいました。

最後に

自身の初めてのポートフォリオが完成しました。

想定よりも苦戦はしてしまいましたが、個人的には出来の良いポートフォリオが完成したのではないかと考えています。

これで終わりにせず、今後もアップデートを重ねていく予定です。

以上で紹介を終わりとさせていただきます。

ご覧いただきありがとうございました。
